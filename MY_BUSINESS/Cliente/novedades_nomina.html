1. Datos generales de la novedad

Empleado y período Según el tipo seleccionado aparecerán los campos específicos en las secciones siguientes. Para novedades de un solo día dejar igual a fecha de inicio. Para cambios de condición (salario, cargo) no aplica fecha fin. Se calcula automáticamente
Descripción y soporte Formatos: PDF, JPG, PNG. Tamaño máximo: 5MB. Sin soporte la novedad no podrá ser aprobada. Los soportes se conservan 20 años por normativa UGPP.

2. Incapacidades

Complete esta sección solo si el tipo de novedad es incapacidad por enfermedad general o accidente laboral.

Datos de la incapacidad Para accidentes de trabajo: número del FURAT (Formulario Único de Reporte de Accidentes de Trabajo). Sin este número la ARL puede negarse a reconocer el pago. Máximo legal: 180 días prorrogables hasta 540 días (Art. 227 CST + Decreto 2943/2013). Si supera 180 días → la empresa puede iniciar proceso de pérdida de capacidad laboral. Se calcula automáticamente Se calcula automáticamente Se calcula automáticamente El código CIE-10 es obligatorio para recobros a la EPS. Sin este código la EPS puede objetar el pago. Si es reincidencia en la misma patología con menos de 30 días de diferencia, se cuenta como una sola incapacidad continua para efectos del pago.

3. Vacaciones

Complete esta sección solo si el tipo de novedad es vacaciones disfrutadas o compensadas.

Control de vacaciones Se carga al seleccionar el empleado Mínimo recomendado: 6 días continuos (para que sean efectivamente descanso). Máximo por período: 15 días hábiles. Recuerde que los festivos que caigan dentro de las vacaciones no cuentan como días de vacaciones. Se calcula automáticamente sumando los días aprobados más festivos que caigan dentro del período. Reglas de cálculo y estructura de base de datos

Tabla: novedades_nomina

Estructura de la tabla principal de novedades
Campo Tipo Descripción Regla crítica
id_novedad INT PK autoincrement Identificador único —
id_empleado INT FK → empleados Empleado afectado No puede estar en estado "retirado"
id_periodo INT FK → periodos_nomina Período al que aplica Solo períodos con estado "abierto"
tipo_novedad ENUM Clasificación de la novedad Determina qué campos son obligatorios
fecha_inicio DATE Inicio de la novedad No puede ser anterior a fecha de ingreso
fecha_fin DATE NULL Fin de la novedad NULL para cambios de condición
dias_novedad DECIMAL(5,2) Días calculados Calculado por el sistema, no digitado
valor_afecta_empleado DECIMAL(15,2) Valor que descuenta al empleado Positivo = descuento, negativo = pago adicional
valor_afecta_empresa DECIMAL(15,2) Costo o ahorro para la empresa Para control de provisiones y presupuesto
valor_recobro_eps_arl DECIMAL(15,2) Lo que debe reintegrar la EPS o ARL Se lleva a cuentas por cobrar en contabilidad
afecta_ibc BOOLEAN ¿Modifica la base de cotización? Crítico para seguridad social
ruta_soporte VARCHAR(500) Ruta del documento soporte Obligatorio para aprobar la novedad
estado_novedad ENUM borrador/pendiente/aprobada/rechazada/procesada Solo "aprobada" se incluye en la nómina
id_aprobador INT FK → empleados Quién aprobó la novedad Obligatorio antes de procesar
fecha_aprobacion DATETIME Cuándo fue aprobada Para trazabilidad UGPP
usuario_registro INT FK → usuarios Quién la digitó Auditoría obligatoria
fecha_registro DATETIME Cuándo fue digitada No modificable después de guardar

Tabla: acumulados_empleado

Esta tabla es el "saldo de cuenta" de cada empleado. El sistema la actualiza automáticamente con cada novedad procesada. Sin ella no puedes saber cuántos días de vacaciones le quedan a un empleado ni cuál es su saldo de cesantías acumulado.

Acumulados por empleado — se actualiza con cada nómina
